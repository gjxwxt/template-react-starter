# Architecture

## 目录分层

```text
src/
  api/          请求客户端、接口模块、mock 分发
  app/          全局运行时、路由、会话、导航、配置
  components/   跨域通用组件
  layouts/      应用壳层
  modules/      跨页面复用的业务域逻辑
  pages/        路由页面入口
  styles/       全局样式入口和模板级样式规则
```

更细的目录拆分规则、页面与模块的边界、`store` 与 `utils` 的落位策略，统一以 `PROJECT_STRUCTURE.md` 为准。

## 核心运行链路

### 1. 登录链路

1. 页面调用 `src/api/auth.ts`
2. 请求层走 `src/api/client.ts`
3. mock 模式下由 `src/api/mock/auth.ts` 生成最小登录态
4. 会话写入 `src/app/session/storage.ts`
5. 壳层读取会话并决定首屏可访问页面

### 2. 壳层上下文链路

1. `src/layouts/app-shell.tsx` 负责整个应用壳层
2. 若 `navigationMode=server`，壳层会在登录后请求 `/app/context`
3. 上下文返回资料、可访问路由、菜单分组、权限码
4. 壳层只更新会话中的壳层字段，不重建登录态

### 3. 请求层链路

1. `src/api/client.ts` 统一处理 token、超时、mock / real 切换
2. 请求层兼容常见企业接口包裹：`code/msg/data/success`
3. 401 会广播登录失效
4. 403 / 404 / 500 会跳转到路由级异常页

## 页面样例分层

- 标准列表页：任务列表、用户管理
- 权限树页：角色管理
- 左树右表页：部门管理
- 主从表页：字典管理

### 左树右表页默认策略

- 桌面端优先使用“左侧 sticky 树过滤 + 右侧主内容滚动”的布局，不让整页跟着树一起拉长
- 左树节点默认展开 1 到 2 层，先给用户结构上下文，不默认把整棵树全部炸开
- 当树节点超出可视区时，优先让树区域内部滚动，右侧仍然保持列表和表格作为主操作区

## 关键设计

### 本地路由是真源

页面注册只放在 `src/app/router/routes.tsx`。这样可以稳定控制：

- 页面组件引用
- 默认首页和 affix 页签
- 兜底跳转
- 页面级角色限制

### 服务端菜单只改“壳层表现”

服务端菜单模式不会创建未知页面，它只影响：

- 当前账号能看到哪些页面
- 侧边栏如何分组和排序
- 菜单文案如何展示
- 页面级操作是否拥有 `permissionCodes`

### mock 与真实 API 共用一套消费模型

只要后端最终能落成壳层上下文模型，页面和壳层就不需要再关心“这是 mock 还是线上接口”。
