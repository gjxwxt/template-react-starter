# Product

## 模板定位

`template-react` 是基于 `@gjxwxt/react-components` 的企业后台起步模板。它的目标是让业务项目拿到仓库后，优先关注业务接口和业务页面，而不是从零搭壳层、权限、页面骨架。

## 目标用户

- 需要快速启动中后台项目的前端团队
- 需要给业务方提供 React 模板的组件库团队
- 需要稳定接力开发的 AI / 人类协作团队

## 当前提供的核心能力

- 登录页与本地会话持久化
- 应用壳层、侧边栏、顶部操作区、多页签导航
- 品牌 / 浅色 / 深色三套侧边栏主题
- Dashboard、列表页、详情页、系统管理页样例
- 用户、角色、部门、字典四类高频系统管理母版
- mock / real API 双模式请求层
- 403 / 404 / 500 异常页
- 本地路由模式 / 服务端菜单模式双通道
- 个人中心与壳层上下文基础能力

## 设计目标

1. 开箱即用：能直接启动、演示、改造
2. 企业级：壳层、菜单、权限、异常页、表单表格组合要完整
3. 可接入：对真实后端、真实菜单树、真实错误码有明确适配点
4. 可维护：AI 和团队成员都能快速读懂关键路径

## 明确边界

- 不把模板做成“后端返回任何 URL 都能自动生成页面”的运行时平台
- 不把业务项目的特殊约束直接固化成模板默认行为
- 不默认承诺组织级 CLI 生成器已经成熟可发版
- 不把所有全局主题能力一次性做满，当前重点仍是壳层和页面母版

## 下一阶段高价值方向

- 模板级 smoke / E2E
- 更广泛地把 `permissionCodes` 延伸到任务列表等业务页
- 真实后端接入样例
- 在试点项目稳定后，再评估是否抽成命令式生成器
